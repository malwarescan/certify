# P1.5 "One Plate + Mechanism" — Dev Directive

**Goal:** Header feels physically part of the hero plate; seal motion reads as precise instrument, not floating sticker.

---

## A) HEADER: MERGE INTO HERO PLATE (P0)

### 1. Remove the "bar edge" feeling
- Do NOT use a distinct border-bottom line on the header.
- One shared hero plate background spanning under header AND hero.

**Implementation:**
- `.hero-plate` wrapper contains Header + Hero.
- Plate background applied on hero wrapper, not separately to header.

### 2. Header film: "thin lacquer," not "frost panel"
- Blur: 8–10px (not 12px)
- Saturate: 1.05–1.10
- Inset highlight alpha: 0.35–0.45 (not 0.65)
- Bottom gradient: 0.78 → 0.60

### 3. Nav weight + spacing
- font-medium
- Desktop: gap-5 (gap-4 feels cramped)

**Acceptance:** At 1440px wide, header feels like it's floating on the same sheet as the hero.

---

## B) SEAL: STOP "DRIFT" — MAKE IT "MECHANISM" (P0)

### 1. Remove seal-drift translation completely
- Kill translateY motion. Keep rotation mechanisms only.
- Outer rotor: constant rotation
- Inner rotor: reverse rotation
- Specular: sweep
- Glow breathe: opacity only
- Optional: 0.15–0.25deg wobble on rotor only (NOT whole seal)

### 2. Notch as "movement witness"
- Notch opacity floor 0.65
- Crisp, always above overlays

### 3. One-time activation event
- IntersectionObserver: add `data-seal-active="1"` once when seal enters viewport.
- CSS keyframe: 600–800ms ring ripple, opacity 0.12 max.

**Acceptance:** No perpetual vertical movement. Movement via notch + specular, not floating.
