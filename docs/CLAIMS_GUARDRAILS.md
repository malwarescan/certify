# CLAIMS GUARDRAILS — CERTIFICATE.NOW

**Purpose:** Legal + trust. When specific terms may be used.

---

## BLOCKCHAIN LANGUAGE

| Term | When Allowed | When Not Allowed |
|------|--------------|------------------|
| **Blockchain secured** | Never as hero/eyebrow claim | Only in feature bullets if you can point to actual chain + verify entry |
| **Registry-backed** | Preferred umbrella term | — |
| **Anchored** | When blockchain exists; must display tx reference in details | Without verifiable tx |
| **Blockchain** | In Details drawer, Evidence section, with tx hash/link | Marketing copy, hero, CTA |

**Rule:** Do not claim "blockchain secured" unless you can point to the actual chain and verify the entry.

---

## REGISTRY LANGUAGE

- **Registry-backed** — Safe. Use for infrastructure positioning.
- **Registry entry** — Safe. Use when you have a record but don't want to claim blockchain specifics.
- **Ledger** — Use only if you have a real ledger reference to show.

---

## VERIFICATION LANGUAGE

- **Verified** — Only when status = verified and evidence supports it.
- **Authentic** — Same bar as Verified.
- **Not Verified** — When no matching certificate found. Never "Fake" or "Counterfeit" unless legally confirmed.

---

## ENFORCEMENT

- All copy must pass this guardrail before ship.
- Marketing cannot override. If in doubt, use "registry-backed" and link to details.
